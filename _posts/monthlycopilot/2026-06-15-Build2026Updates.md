<h2 class="mc-section-title">AI 에이전트 플랫폼의 진화</h2>

<p style="color: #1d4ed8; font-weight: 700; letter-spacing: 0.05em; margin-bottom: 0.8rem;">FRONTIER DEVELOPMENT</p>

<p>소프트웨어 개발이 큰 전환점을 맞고 있습니다. AI 에이전트는 단순히 거드는 도구를 넘어, 복잡한 일을 스스로 오랫동안 수행하는 <strong>장기 실행(long-running)</strong> 시스템으로 발전하고 있습니다. 디지털 동료로서의 에이전트를 경쟁력 있는 솔루션으로 키우려면, 풍부한 맥락을 담은 지능(Intelligence)과 신뢰성(Trust)을 함께 갖춰야 합니다.</p>

<p>이번 Build에서 Microsoft는 인프라와 에코시스템, 디바이스, 에이전트 플랫폼·도구, 모델은 물론, 도메인 특화 추론(Reasoning) 역량까지 폭넓게 선보였습니다. <strong>Microsoft Agent Platform Stack</strong>은 AI를 만들고(build), 연결하고(connect), 실행하고(run), 관리하는(govern) 전 과정을 아우르며, 여러 에이전트가 한 팀처럼 협력해 실제 업무를 처리하도록 돕습니다.</p>

<img src="/mwkorea/assets/images/20260605-build/agent-platform-stack.png" alt="Microsoft Agent Platform Stack" />

<h3>에이전트에 '맥락'을 더하는 IQ 제품군</h3>

<p>모델 성능이 서로 비슷해지는 지금, 기업의 경쟁력은 흩어진 지식을 하나로 모아 에이전트에 연결하는 능력에서 갈립니다. 그 핵심이 <strong>IQ 제품군</strong>입니다.</p>

<div class="mc-card-grid mc-card-grid--3">
  <div class="mc-card mc-card--blue">
    <div class="mc-card-title">Microsoft IQ</div>
    <div>세상의 지식과 기업 내부 지식을 함께 연결하는 컨텍스트 계층. 오늘부터 GitHub Copilot·Microsoft Foundry·Copilot Studio에서 정식 제공됩니다.</div>
  </div>
  <div class="mc-card mc-card--teal">
    <div class="mc-card-title">Work IQ</div>
    <div>사람·이메일·문서·회의와 그 연결 관계까지 읽어 '업무 맥락'을 이해하는 지능 계층. <strong>Work IQ API</strong>는 6월 16일 정식 출시됩니다.</div>
  </div>
  <div class="mc-card mc-card--purple">
    <div class="mc-card-title">Fabric IQ</div>
    <div>매출·재고처럼 표로 정리된 정형 데이터를 에이전트가 일관되게 해석하도록 공통 의미 체계(시맨틱 계층)를 제공합니다.</div>
  </div>
  <div class="mc-card mc-card--amber">
    <div class="mc-card-title">Foundry IQ</div>
    <div>위 요소를 하나로 묶어, 사내 지식과 실시간 웹을 넘나들며 답을 구성하는 검색·추론(retrieval) 과정을 담당합니다.</div>
  </div>
  <div class="mc-card mc-card--green">
    <div class="mc-card-title">Web IQ <span style="font-weight:700;">(신규)</span></div>
    <div>에이전트를 실시간 웹에 가장 빠르게 연결합니다. 모델에 얽매이지 않고(model-agnostic) MCP를 기본 지원하며, 동급 최고 수준보다 2.3배 빠르게 정보를 찾아 줍니다.</div>
  </div>
  <div class="mc-card">
    <div class="mc-card-title">왜 IQ인가</div>
    <div>구성원이 일하는 방식과 사내 곳곳에 흩어진 지식을 에이전트에 연결하는 능력이, 이제 기업 경쟁력의 핵심입니다.</div>
  </div>
</div>

<h3>만들고, 배포하고, 운영하는 도구</h3>

<ul>
<li><strong>GitHub Copilot app</strong> — GitHub Copilot을 데스크톱 앱으로 새롭게 선보였습니다(프리뷰). 아이디어·이슈·PR에서 작업을 시작해 여러 에이전트를 동시에 돌리고, 코드 리뷰부터 CI·병합(merge)까지 하나의 흐름으로 이어집니다. 각 작업은 git worktree 단위로 분리되며, 실행은 Copilot이 맡되 모든 과정은 개발자가 직접 통제합니다.</li>
<li><strong>Rayfin</strong> — 애플리케이션 백엔드를 코드 안에서 정의하고 Microsoft Fabric에 곧장 배포하는 오픈소스 SDK이자 CLI. 인프라를 따로 구성하지 않고도 GitHub 워크플로우만으로 프로토타입에서 실제 서비스까지 끌어올립니다. <strong>Replit</strong>과 협력해, 코딩 에이전트가 만든 프로토타입을 처음부터 거버넌스가 적용된 엔터프라이즈 환경으로 옮길 수 있습니다.</li>
<li><strong>Microsoft Foundry</strong> — 인프라를 직접 관리할 필요 없이 에이전트를 운영하는 <strong>Hosted Agents</strong>를 선보였고, <strong>Fireworks AI</strong>와 협력해 모델 학습·파인튜닝을 Foundry 거버넌스 안에서 수행할 수 있게 했습니다. 모든 과정은 <strong>Foundry Control Plane</strong> 한곳에서 관리됩니다. 정책 기반 안전성 점검 <strong>ASSERT</strong>와 에이전트 통제 표준 <strong>Agent Control Specification</strong> 두 오픈소스도 공개됐습니다.</li>
<li><strong>Azure HorizonDB</strong> — 초저지연 PostgreSQL 기반의 새 데이터베이스. 읽기 부하 분산(read scale-out)과 AI 기능을 내장하고 Azure와 긴밀히 연동돼 미션 크리티컬 서비스를 든든하게 받칩니다. VS Code의 GitHub Copilot, 고급 벡터 인덱싱·시맨틱 검색까지 지원합니다.</li>
</ul>

<h3>안전하게 관리하는 Agent 365</h3>

<div class="mc-card-grid">
  <div class="mc-card mc-card--blue">
    <div class="mc-card-title">Agent 365 SDK</div>
    <div>보안·기업 요건을 갖춘 에이전트를 더 쉽게 만들도록 정식 출시. 개발 단계부터 모니터링·접근 제어·규정 준수가 자동 적용되며, 어떤 AI 플랫폼의 에이전트라도 Agent 365와 자연스럽게 연결됩니다.</div>
  </div>
  <div class="mc-card mc-card--teal">
    <div class="mc-card-title">Agent 365</div>
    <div>Entra·Defender·Purview를 하나의 관리 체계로 묶어 로컬 에이전트까지 모두 한곳에서 관찰·통제·보호합니다. <strong>A365 Agent Registry</strong>가 '관리되지 않는 에이전트'를 찾아내고, 에이전트용 Windows 365와 <strong>Microsoft Execution Container(MXC)</strong>가 기본 보호 장치를 제공합니다.</div>
  </div>
  <div class="mc-card mc-card--red">
    <div class="mc-card-title">MDASH (코드네임)</div>
    <div>100개가 넘는 전문 AI 에이전트를 여러 모델로 함께 돌려 보안을 점검. 취약점을 찾는 데 그치지 않고 실제 악용 가능성까지 검증하며, 결과는 Defender 포털에서 바로 확인합니다.</div>
  </div>
  <div class="mc-card mc-card--purple">
    <div class="mc-card-title">왜 중요한가</div>
    <div>속도를 늦추지 않으면서도, 빠르게 늘어나는 에이전트를 안전하게 관리할 수 있습니다. 그동안 보이지 않던 관리·보안의 사각지대를 없애 줍니다.</div>
  </div>
</div>

<h3>어디서나 작동하는 에이전트 — 디바이스와 런타임</h3>

<p>에이전트 시대에는 클라우드든 디바이스든 어디서나 똑같이 작동하는 인텔리전스 플랫폼이 필요합니다. 개발자에게 가장 중요한 건 강력한 에이전트를 만드는 동시에 주도권을 스스로 갖는 것입니다.</p>

<ul>
<li><strong>Surface RTX Spark</strong> — 모델 학습, 에이전트 AI 파이프라인, 로컬 파인튜닝을 위한 개발자용 디바이스. NVIDIA N1X Arm SoC 기반으로 100W의 낮은 전력에서도 최대 1페타플롭(petaflop) 연산과 128GB 통합 메모리를 제공해, 책상 위에서 700억(70B) 규모 모델까지 직접 돌립니다. CUDA 지원 WSL 2, VS Code·GitHub Copilot이 기본 탑재됐고 올해 하반기 출시 예정입니다.</li>
<li><strong>Microsoft Execution Containers(MXC)</strong> — Windows를 에이전트 구축·실행에 가장 좋은 환경으로 만드는 격리 기술. SDK가 오늘 공개되고 Process·Session 컨테이너가 Windows Insider Program 프리뷰로 제공됩니다. 격리(containment)는 운영체제 자체에서 직접 적용됩니다.</li>
<li><strong>OpenClaw on Windows</strong> — 이미 Windows에 적용되어, OS가 정해 둔 보안 범위 안에서 여러 단계를 거치는 워크플로우를 자동으로 실행합니다.</li>
<li><strong>NVIDIA OpenShell</strong> — 선언형 에이전트 정책을 Windows의 MXC 격리에 매핑해 샌드박싱·파일 권한·네트워크 정책·자격 증명·추론까지 포괄합니다. 개발자에게는 개방형 런타임을, IT에는 일관된 가시성과 통제력을 제공합니다.</li>
<li><strong>Microsoft Scout</strong> (Frontier Preview) — Microsoft가 처음 선보이는 '오토파일럿(autopilot)' 개념의 데스크톱 에이전트. 항상 켜진 상태로 여러 앱·시스템에 걸친 업무 흐름을 이해하고, 매번 지시하지 않아도 필요한 일을 알아서 챙깁니다. Teams·Outlook·OneDrive·SharePoint 등 Microsoft 365 앱에 자연스럽게 통합됩니다.</li>
</ul>

<div class="mc-callout">
  <strong>확장되는 협력</strong> — Windows는 NVIDIA·OpenClaw 외에도 OpenAI, Hermes, AMD 등 업계 선도 기업들과 협력하여, 구축 중인 격리 기술이 개발자의 실제 요구를 충족하도록 하고 있습니다.
</div>

<h2 class="mc-section-title">AI 지능의 확장 — MAI 모델 제품군</h2>

<p style="color: #1d4ed8; font-weight: 700; letter-spacing: 0.05em; margin-bottom: 0.8rem;">FRONTIER OPPORTUNITIES</p>

<p>모델 측면에서는, Microsoft AI Superintelligence Team이 자체 개발한 최신 세대 모델들을 새롭게 공개했습니다.</p>

<div class="mc-card-grid">
  <div class="mc-card mc-card--blue">
    <div class="mc-card-title">MAI-Thinking-1</div>
    <div>Microsoft AI의 첫 텍스트·추론·코딩 모델. 350억 개 활성 파라미터와 128K 컨텍스트를 갖췄고, 적은 비용으로도 높은 성능을 냅니다. 복잡한 다단계 지시·긴 맥락 추론·코드 생성에 강하며, 현재 Microsoft Foundry에서 프라이빗 프리뷰로 제공됩니다.</div>
  </div>
  <div class="mc-card mc-card--teal">
    <div class="mc-card-title">MAI-Image-2.5</div>
    <div>글로 설명하면 이미지를 만들고(text-to-image), 기존 이미지를 새 이미지로 바꾸는(image-to-image) 기능을 모두 갖춘 Microsoft의 첫 이미지 모델. 아이디어 시각화와 결과물 다듬기에 유용합니다.</div>
  </div>
  <div class="mc-card mc-card--purple">
    <div class="mc-card-title">MAI 모델 제품군 확장</div>
    <div><strong>MAI-Transcribe-1.5</strong>(고유 명칭 인식 강화·실시간 스트리밍 예정), <strong>MAI-Voice-2</strong>(10개 이상 언어), <strong>MAI-Code-1</strong>(Flash·Pro, GitHub Copilot·VS Code 최적화)이 합류. Microsoft Foundry와 MAI Playground에서 바로 사용 가능합니다.</div>
  </div>
  <div class="mc-card mc-card--amber">
    <div class="mc-card-title">Frontier Tuning</div>
    <div>회사 데이터와 업무 방식을 학습 재료로, 규정 안에서 강화학습(RL)으로 AI를 꼭 맞게 길들이는 방식. 전담 엔지니어(FDE)를 통해 프라이빗 프리뷰로 제공되며 Copilot Studio·Microsoft Foundry에도 곧 등장합니다.</div>
  </div>
</div>

<h2 class="mc-section-title">차세대 혁신</h2>

<p style="color: #1d4ed8; font-weight: 700; letter-spacing: 0.05em; margin-bottom: 0.8rem;">NEW FRONTIERS</p>

<p>장기 실행 에이전트가 소프트웨어 개발과 개발자의 역할을 바꿔 놓았듯, 이제 새로운 에이전트가 연구개발(R&D)과 과학자가 일하는 방식까지 바꾸기 시작했습니다.</p>

<div class="mc-card-grid">
  <div class="mc-card mc-card--green">
    <div class="mc-card-title">Microsoft Discovery <span style="font-weight:700;">(GA)</span></div>
    <div>Foundry와 GitHub Copilot 기반으로, 가설 수립·실험·결과 분석까지 연구의 전 과정을 에이전트가 돕는 플랫폼. BHP는 구리 추출(copper-leaching) 공정을 몇 달 만에 찾고, Syensqo는 반도체 냉각, GSK는 신약 개발을 앞당기고 있습니다. 개인 PC에 설치하는 무료 버전도 제공됩니다.</div>
  </div>
  <div class="mc-card mc-card--blue">
    <div class="mc-card-title">Majorana 2</div>
    <div>차세대 양자 프로세서. 이전 세대 Majorana 1보다 큐비트 안정성을 1,000배 높였고, 소재를 알루미늄에서 납으로 바꾼 새 구조를 적용했습니다. 신용카드보다 작은 칩 하나에 큐비트 100만 개를 담을 길이 열렸습니다.</div>
  </div>
</div>

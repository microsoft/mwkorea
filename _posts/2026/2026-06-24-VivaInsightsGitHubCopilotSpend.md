# GitHub Copilot 지출·사용 현황을 한눈에: Viva Insights에 AI 비용 인사이트 추가

AI 도구를 조직에 도입한 뒤 따라오는 질문은 늘 비슷합니다. **"실제로 얼마나 쓰고 있고, 비용은 어디에 들어가고 있는가?"** 특히 개발 조직에서 GitHub Copilot 사용이 늘어날수록, 라이선스 지출과 실제 활용도를 연결해 보는 일이 점점 중요해집니다.

Microsoft는 Microsoft 365 Roadmap을 통해, **Viva Insights에 GitHub Copilot의 AI 지출(spend) 및 사용(usage) 인사이트**가 추가된다고 공지했습니다. 관리자와 분석가가 GitHub Copilot의 비용과 활용을 추적하고, **데이터에 근거한 AI 투자 의사결정**을 내릴 수 있도록 돕는 대시보드입니다.

무엇보다 이 대시보드는 **기본적으로 활성화(enabled by default)**되어, 대상자가 즉시 접근할 수 있다는 점이 특징입니다.

---

## 무엇이 추가되나

- **GitHub Copilot 지출·사용 인사이트**: Viva Insights 안에서 GitHub Copilot의 AI 지출과 사용 현황을 확인할 수 있습니다.
- **데이터 기반 의사결정**: 단순 집계를 넘어, AI 지출과 활용을 함께 보며 투자 우선순위와 라이선스 배분을 판단하는 데 활용할 수 있습니다.
- **기본 활성화**: 대시보드가 기본적으로 켜져 있어, 대상 사용자가 별도 준비 없이 바로 접근할 수 있습니다.

---

## 누가 볼 수 있나 (접근 권한)

이 인사이트는 모든 사용자에게 열려 있지 않고, 역할에 따라 **범위가 제한(scoped access)**됩니다.

| 대상 | 접근 범위 |
|------|-----------|
| **직속 부하 5명 이상 관리자** | 범위 제한 접근(scoped access) — 자신의 조직 범위 내 |
| **글로벌 분석가(Global analysts)** | 분석 권한 범위 |
| **Microsoft 글로벌 관리자(Global administrators)** | 전체 |

> 관리자의 경우 **직속 부하 5명 이상**이라는 최소 조건이 있어, 소규모 단위에서 개인이 식별되는 것을 방지하는 집계·프라이버시 보호 장치가 함께 적용됩니다.

---

## 왜 중요한가

- **AI 비용 가시성(Visibility)**: GitHub Copilot 같은 AI 도구는 도입은 쉽지만, 비용 대비 효과를 추적하기는 어렵습니다. 지출과 사용을 한 화면에서 보면 "투자가 실제 활용으로 이어지는지"를 점검할 수 있습니다.
- **거버넌스와 최적화**: 사용이 저조한 라이선스를 찾아 재배분하거나, 활발한 팀에 자원을 더 배치하는 등 **데이터 기반 최적화**가 가능해집니다.
- **재무·IT·리더십의 공통 언어**: 동일한 대시보드를 기준으로 대화하면, AI 투자에 대한 부서 간 논의가 추측이 아닌 근거 위에서 이루어집니다.

---

## 출시 일정

| 항목 | 내용 |
|------|------|
| 기능 | Viva Insights 내 GitHub Copilot 지출·사용 인사이트 대시보드 |
| 접근 | 직속 부하 5명 이상 관리자(범위 제한), 글로벌 분석가, 글로벌 관리자 |
| 활성화 | 기본 활성화(enabled by default) |
| 정식 출시(GA) | **2026년 7월 (예정)** |

---

## 도입 관점 체크포인트

- **기본 활성화 인지**: 대시보드가 기본으로 켜지므로, 대상 관리자·분석가에게 "어떤 데이터가, 누구에게 보이는지"를 사전에 안내해 두면 좋습니다.
- **프라이버시 정책 검토**: 직원 단위 데이터가 포함될 수 있으므로, 사내 프라이버시·데이터 거버넌스 정책과 어긋나지 않는지 확인하고 접근 범위를 점검하세요.
- **AI 비용 관리 프로세스와 연계**: 기존 라이선스·비용 관리 프로세스(FinOps, IT 자산관리 등)와 이 인사이트를 어떻게 연결할지 정리해 두면, 단발성 조회를 넘어 지속적인 최적화 루틴으로 이어집니다.
- **GitHub Copilot 운영 팀과 공유**: 개발 조직의 GitHub Copilot 운영 담당과 데이터를 함께 보며 활용 패턴을 해석하면, 숫자에 맥락이 더해집니다.

---

## 마무리

이번 업데이트는 "AI를 도입했는가"에서 "**AI를 제대로, 비용 효율적으로 쓰고 있는가**"로 질문을 옮겨 줍니다. Viva Insights가 생산성 분석을 넘어 **AI 투자 관리의 관제탑** 역할까지 확장하는 흐름의 한 단계로, GitHub Copilot을 도입한 조직이라면 출시 시점에 맞춰 대시보드 접근 권한과 활용 프로세스를 미리 정비해 두기를 권합니다.

# Microsoft 365 Copilot 사용량 보고서, GCC High·DoD 환경까지 확대됩니다

상업용(Commercial) 테넌트에서는 이미 익숙한 **Microsoft 365 Copilot 사용량 보고서**가, 이번 업데이트로 **GCC High·DoD** 환경까지 확대 제공됩니다. 그동안 이 두 환경의 관리자는 Copilot 라이선스를 배포하고도 실제 활용도를 파악할 표준 보고서가 없어 답답함을 느꼈을 수 있는데, 이번 로드맵 공지로 그 간극이 좁혀집니다.

> **이 글의 한 줄 요약**
> Microsoft 365 admin center의 Usage reports에서 GCC High·DoD 환경에도 Microsoft 365 Copilot 보고서가 제공됩니다. 활성화(enabled)·활성(active) 사용자 수, 앱별 사용 현황 breakout, 사용자 단위 활동 인사이트(기본 익명화)를 확인할 수 있으며, 출시는 **2026년 7월** 예정입니다.

---

## 무엇이 새로워지나

이번 업데이트의 핵심은 **범위 확대**입니다. 상업용 테넌트에서 제공되던 Microsoft 365 Copilot 사용량 보고서가 규제 수준이 더 높은 GCC High·DoD 환경까지 동일하게 적용됩니다. 앞서 소개된 Copilot Chat 전용 보고서(GCC·GCC High·DoD 대상)와 함께, admin center의 Copilot 관련 리포팅 체계가 정부·국방 고객 대상으로 한층 촘촘해지는 흐름입니다.

## 핵심 기능

로드맵 공지에 명시된 보고서 구성 요소는 다음과 같습니다.

- **전체 활성화·활성 사용자 수(Total enabled and active users)**: 라이선스가 부여된 인원과 실제 사용 중인 인원을 함께 파악
- **앱별 사용 현황 breakout**: Microsoft 365 앱별로 Copilot 사용 패턴을 나누어 확인
- **사용자 단위 활동 인사이트**: 개인별 활동 데이터를 제공하되 **기본값은 익명화(anonymized by default)**

이 역시 `#copilotcontrolsystem` 태그가 붙어 있어 Copilot 거버넌스·관리 체계의 연장선에 있는 업데이트임을 알 수 있습니다.

## 활용 시나리오

- **라이선스 활용률 점검**: enabled 대비 active 사용자 비율을 확인해 유휴 라이선스 여부를 조기에 파악
- **부서·업무별 확산 전략 수립**: 앱별 사용 편차를 근거로 교육·챔피언 프로그램을 어디에 집중할지 결정
- **컴플라이언스 대응**: DoD·GCC High 특유의 감사·보고 요건에 맞춰 익명화된 사용 데이터를 근거 자료로 활용

## 출시 일정

| 구분 | 내용 |
|------|------|
| **대상 환경** | GCC High, DoD |
| **위치** | Microsoft 365 admin center → Usage reports |
| **GA(정식 출시)** | 2026년 7월 (CY2026) |

## 도입 관점 체크포인트

- GCC High·DoD 환경 관리자 권한 범위와 이 보고서에 대한 접근 권한이 일치하는지 확인
- 상업용 테넌트와 별도로 운영되는 경우, 두 환경의 사용량 리포팅 주기·양식을 통일할지 결정
- 사용자 단위 데이터 활용 시 기관 내 개인정보·보안 정책과의 정합성 재확인

## 마무리

Copilot Chat에 이어 Microsoft 365 Copilot 전체 사용량 보고서까지 GCC High·DoD로 확대되면서, 정부·국방 고객도 상업용 고객과 동등한 수준의 데이터 기반 도입 관리가 가능해집니다. 2026년 7월 GA 이후 admin center에서 확인해 보시기 바랍니다.

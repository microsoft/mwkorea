# 스캔한 계약서도, 이미지 속 표도 검색된다: Copilot Chat의 텍스트 인식 검색 강화

회사 안에는 텍스트로 저장되지 않은 정보가 생각보다 많습니다. 스캔해서 PDF로 보관한 옛날 계약서, 회의 중 찍은 화이트보드 사진, 다른 곳에서 캡처해 붙여넣은 표 스크린샷. 사람 눈에는 분명히 읽히는 정보인데, 지금까지 검색으로는 좀처럼 찾아지지 않았습니다.

Microsoft 365 Roadmap 항목 RM559613에 따르면, **Copilot Chat**이 이제 **스캔한 PDF**와 **Word·Excel·PowerPoint에 삽입된 이미지 속 텍스트**까지 검색 결과에 더 잘 매칭하도록 개선됩니다. Word의 스캔 계약서, Excel의 표 스크린샷, PowerPoint의 화이트보드 사진처럼 지금까지 찾기 어려웠던 콘텐츠가 검색 대상이 됩니다.

---

## 무엇이 새로워지나

- **스캔 PDF 검색 매칭**: 이미지로만 존재하던 스캔 PDF의 텍스트를 인식해 검색 결과에 반영
- **문서 내 삽입 이미지 텍스트 인식**: Word·Excel·PowerPoint 파일 안에 붙여넣거나 삽입한 이미지 속 텍스트까지 검색 대상에 포함
- **대표 시나리오**: Word의 스캔 계약서, Excel의 표 스크린샷, PowerPoint의 화이트보드 사진

---

## 왜 중요한가

조직의 지식은 깔끔한 텍스트 파일로만 존재하지 않습니다. 오래된 계약서, 회의 중 찍은 사진, 다른 시스템에서 캡처한 화면 — 이런 자료는 실제로 중요한 정보를 담고 있지만, 텍스트 기반 검색에서는 '보이지 않는' 콘텐츠였습니다.

이번 개선은 Copilot Chat이 **이미지 속 텍스트까지 이해하고 검색에 반영**하도록 해, 사용자가 "그 계약서 어디 있더라", "그때 찍은 화이트보드 사진 어디 있지"라는 질문에 더 나은 답을 받을 수 있게 합니다. 문서를 다시 타이핑하거나 별도로 정리할 필요 없이, 있는 그대로의 자료가 검색 가능한 지식이 되는 셈입니다.

---

## 활용 시나리오

- **오래된 계약서 검색**: 스캔해서 보관해 온 계약서를 키워드로 검색해 빠르게 찾기
- **회의 자료 재발견**: 화이트보드 사진, 손글씨 메모 스캔본에서 특정 내용 검색
- **데이터 스크린샷 활용**: 다른 시스템에서 캡처해 붙여넣은 표·차트 이미지도 검색 대상으로 포함
- **아카이브 문서 활용도 향상**: 그동안 사실상 검색 사각지대였던 이미지 기반 문서의 활용도 증가

---

## 도입 관점 체크포인트

- **아카이브 문서 점검**: 조직에 스캔 문서·이미지 기반 자료가 많다면, 이번 기능으로 검색 활용도가 크게 오를 수 있습니다. 어떤 자료가 검색 가치가 높은지 미리 파악해 두면 좋습니다.
- **권한 범위 재확인**: 검색 가능 범위가 넓어지는 만큼, 기존 파일 접근 권한과 민감정보 보호 정책이 이미지 콘텐츠에도 동일하게 적용되는지 확인하세요.
- **사용자 안내**: "이미지 속 텍스트도 이제 검색된다"는 점을 안내하면, 사용자가 검색 습관을 자연스럽게 넓혀 활용할 수 있습니다.

---

## 마무리

이번 업데이트는 화려하지 않지만 실용적인 개선입니다. 텍스트로 다시 옮기지 않은 채 방치돼 있던 스캔 문서와 이미지 속 정보가, Copilot Chat 검색을 통해 비로소 '찾을 수 있는' 지식이 됩니다. 조직의 숨은 자산을 다시 꺼내 쓰는 계기가 될 만한 업데이트입니다.

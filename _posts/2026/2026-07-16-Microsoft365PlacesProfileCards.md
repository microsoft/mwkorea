# Copilot Chat에서 건물·회의실·좌석 정보를 바로 확인합니다

하이브리드 업무 환경에서는 사람뿐 아니라 **어디에서 일하고 만날지**에 대한 정보도 중요합니다. 특정 회의실이 어느 건물에 있는지, 예약하려는 좌석의 위치가 어디인지 확인하려고 여러 화면을 오가면 작은 마찰이 계속 쌓입니다.

Microsoft Places 디렉터리에 등록된 건물, 회의실, 좌석에 새로운 Profile Card가 제공됩니다.

> **한 줄 요약**
> Microsoft Places의 공간 정보를 Copilot Chat과 여러 Microsoft 365 진입점에서 Profile Card로 바로 확인할 수 있습니다.

---

## 무엇이 새로워지나

Places Directory에 추가된 다음 공간 개체에 Profile Card가 생깁니다.

- Buildings
- Rooms
- Desks

이 카드는 Copilot Chat뿐 아니라 Microsoft 365의 다른 진입점에서도 접근할 수 있습니다.

## 왜 중요한가

Profile Card는 특정 공간에 대한 정보를 하나의 일관된 화면으로 제공하는 기반입니다. 사용자는 Copilot과 대화하거나 Microsoft 365에서 업무를 진행하는 도중 공간 정보를 별도 디렉터리에서 다시 찾지 않아도 됩니다.

공지에는 카드에 표시되는 필드와 지원되는 모든 진입점이 구체적으로 나열돼 있지 않으므로, 실제 제공 범위는 제품 문서와 테넌트에서 확인해야 합니다.

## 활용 시나리오

- Copilot Chat에서 언급된 회의실의 위치 정보를 바로 확인합니다.
- 여러 건물에 걸친 조직에서 공간 이름과 실제 위치를 빠르게 연결합니다.
- Desk 예약이나 출근 계획을 세울 때 Places에 등록된 공간 정보를 확인합니다.
- Microsoft 365 전반에서 같은 공간 정보를 일관되게 제공합니다.

## 출시 일정

| 구분 | 내용 |
|---|---|
| GA | 2026년 6월로 안내 |
| 데이터 원본 | Microsoft Places Directory |
| 대상 | Buildings, Rooms, Desks |
| 접근 위치 | Copilot Chat 및 기타 Microsoft 365 진입점 |

## 도입 관점 체크포인트

- Places Directory의 건물·회의실·좌석 데이터가 최신인지 확인합니다.
- 공간 이름과 위치 체계를 조직 전체에서 일관되게 정리합니다.
- 오래되거나 중복된 공간 개체를 정리해 잘못된 카드 노출을 줄입니다.
- Profile Card에서 사용자에게 표시되는 필드와 개인정보·보안 영향을 점검합니다.

## 마무리

건물·회의실·좌석 Profile Card는 Copilot이 사람과 문서뿐 아니라 실제 업무 공간의 맥락까지 연결하는 변화입니다. 기능을 효과적으로 활용하려면 Places Directory의 데이터 품질을 먼저 관리해야 합니다.

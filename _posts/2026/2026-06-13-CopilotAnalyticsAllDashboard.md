# Copilot 분석, 이제 '전체 사용자'를 한눈에 — Copilot Dashboard 통합 도입 현황

조직에서 Copilot 도입을 추진하는 담당자라면 "우리 조직에서 Copilot이 실제로 얼마나 쓰이고 있나"를 늘 궁금해합니다. 그런데 그동안 **Copilot Dashboard**의 도입 현황(adoption) 랜딩 페이지는 기본적으로 **Microsoft 365 Copilot(라이선스 보유) 사용량만** 보여줬습니다. 라이선스 없이 쓰는 Copilot Chat 사용은 이 기본 화면에 빠져 있었던 셈입니다.

이번 업데이트로 이 랜딩 페이지가 **조직 전체의 Copilot 도입을 통합한 뷰(unified view)**로 바뀝니다. 즉, Microsoft 365 Copilot과 Copilot Chat 사용을 모두 포함해, 한 화면에서 전체 Copilot 채택 현황을 파악할 수 있게 됩니다.

또한 **License 필터**가 추가되어, 보는 사람이 원하는 관점으로 손쉽게 전환할 수 있습니다.

---

## 무엇이 바뀌나

- **통합된 도입 현황 뷰**: Copilot Dashboard 도입 랜딩 페이지가 Microsoft 365 Copilot과 Copilot Chat 사용을 합쳐 조직 전체의 Copilot 채택을 한눈에 보여줍니다.
- **라이선스 없는 사용까지 포함**: 기존에는 빠져 있던, 라이선스가 필요 없는 Copilot Chat 사용 현황도 기본 뷰에 반영됩니다.
- **License 필터 추가**: 다음 세 가지 관점으로 전환할 수 있습니다.
  - **All** — 전체 Copilot 사용자
  - **M365 Copilot (licensed)** — 라이선스 보유 사용자
  - **Copilot Chat (unlicensed)** — 라이선스 없는 Copilot Chat 사용자

## 왜 중요한가

라이선스 기반 사용량만 보면, 실제로 조직에서 일어나고 있는 Copilot 활용의 일부만 보게 됩니다. 특히 Copilot Chat은 별도 라이선스 없이도 쓸 수 있어, 도입 초기 조직에서는 이 영역의 사용이 의외로 큰 비중을 차지하기도 합니다. 통합 뷰는 다음과 같은 점에서 유용합니다.

- **전체 채택 흐름 파악**: 라이선스 보유 여부와 무관하게 조직 내 Copilot 활용 전반을 측정할 수 있습니다.
- **확산 전략 수립**: 라이선스 없는 Copilot Chat 사용이 활발한 부서를 식별해, Microsoft 365 Copilot 라이선스 확대의 근거로 삼을 수 있습니다.
- **보고의 정확도 향상**: 경영진 보고 시 "전체 vs 라이선스 vs 비라이선스"를 명확히 구분해 제시할 수 있습니다.

## 도입 관점 체크포인트

- **기본 뷰 변화 인지**: 기본 화면이 "라이선스 사용량"에서 "전체 사용량"으로 바뀌므로, 기존 수치와 비교할 때 기준이 달라졌음을 염두에 둬야 합니다.
- **필터 활용 가이드**: 대시보드를 보는 담당자들에게 License 필터(All / Licensed / Unlicensed)의 의미를 안내하면 해석 혼선을 줄일 수 있습니다.

## 마무리

이번 변경은 Copilot 분석이 **"라이선스 중심"에서 "조직 전체 활용 중심"으로** 시야를 넓히는 의미 있는 업데이트입니다. 라이선스 없는 Copilot Chat 사용까지 한눈에 들어오면서, 도입 담당자는 조직의 실제 Copilot 채택 수준을 더 정확하게 측정하고 확산 전략을 세울 수 있게 됩니다.

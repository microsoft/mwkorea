# GCC·GCC High·DoD 환경에도 Copilot Chat 전용 사용량 보고서가 추가됩니다

미국 정부기관·방위산업 고객이 사용하는 **GCC(Government Community Cloud), GCC High, DoD** 환경은 상업용(Commercial) 테넌트와 별도의 규제·보안 요건을 따르다 보니, 관리자용 리포팅 기능이 상업용 대비 늦게 제공되는 경우가 많았습니다. Copilot Chat 도입 현황을 파악하고 싶어도 마땅한 사용량 보고서가 없었던 것도 그 중 하나입니다.

이번 Microsoft 365 로드맵 업데이트로 이 공백이 채워집니다. **Microsoft 365 admin center**에 **Copilot Chat만을 위한 전용 사용량 보고서**가 새로 추가되어, GCC·GCC High·DoD 관리자도 상업용 테넌트 관리자와 마찬가지로 도입 현황을 데이터로 확인할 수 있게 됩니다.

> **이 글의 한 줄 요약**
> Microsoft 365 admin center의 Usage reports에 Copilot Chat 전용 보고서가 추가되어, GCC·GCC High·DoD 환경에서도 전체 활성 사용자 수, 앱별 사용 현황, 사용자 단위 활동 인사이트(기본적으로 익명화)를 확인할 수 있게 됩니다. 출시는 **2026년 7월** 예정입니다.

---

## 무엇이 새로워지나

지금까지 GCC·GCC High·DoD 관리자는 Copilot Chat이 조직 내에서 얼마나, 어떻게 쓰이고 있는지 한눈에 볼 수 있는 표준 보고서가 부족했습니다. 이번 업데이트로 admin center의 **Usage reports** 메뉴에 Copilot Chat 전용 보고서가 신설되어, 도입 이후의 활용도를 정기적으로 추적할 수 있는 기반이 마련됩니다.

## 핵심 기능

로드맵 공지에 따르면 새 보고서에는 다음 내용이 포함됩니다.

- **전체 활성 사용자 수(Total active users)**: 조직 전체에서 Copilot Chat을 실제로 사용하고 있는 인원 규모 파악
- **앱별 사용 현황 breakout**: Microsoft 365 앱별로 Copilot Chat 사용 패턴을 구분해서 확인
- **사용자 단위 활동 인사이트**: 개인별 활동 데이터를 제공하되, **기본값은 익명화(anonymized by default)** 처리되어 개인정보 보호를 함께 고려

공지에는 `#copilotcontrolsystem` 태그가 붙어 있어, 이번 보고서가 Copilot 거버넌스·제어 체계의 일부로 다뤄지고 있음을 알 수 있습니다.

## 활용 시나리오

- **도입 효과 측정**: 라이선스를 배포한 부서·팀 단위로 실제 활성 사용률을 확인해 ROI 논의에 활용
- **앱별 온보딩 우선순위 결정**: 어떤 Microsoft 365 앱에서 Copilot Chat 사용이 저조한지 파악해 교육·안내 리소스를 집중
- **거버넌스 리포팅**: 정부기관·방위산업 특유의 컴플라이언스 보고 체계에 맞춰, 익명화된 사용 데이터를 근거 자료로 제출

## 출시 일정

| 구분 | 내용 |
|------|------|
| **대상 환경** | GCC, GCC High, DoD |
| **위치** | Microsoft 365 admin center → Usage reports |
| **GA(정식 출시)** | 2026년 7월 (CY2026) |

## 도입 관점 체크포인트

- 관리자 권한 보유자 중 누가 이 보고서에 접근할 수 있는지 **역할 기반 접근 범위**를 사전에 점검
- 사용자 단위 데이터가 필요한 경우 익명화 해제 조건과 사내 개인정보 처리 방침이 상충하지 않는지 확인
- 기존 상업용 테넌트의 Copilot 사용량 리포팅 프로세스와 GCC 계열 환경의 보고 주기를 맞출지 여부 결정

## 마무리

정부기관·방위산업처럼 별도 클라우드 환경을 쓰는 조직도 이제 Copilot Chat 도입 효과를 데이터로 설명할 수 있는 도구를 갖추게 되었습니다. 2026년 7월 GA 이후 admin center에서 바로 확인해 보시길 권합니다.
